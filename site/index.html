DABstep

Judging data results is hard.

The scorer itself, annotated

def compare_numeric(num1: float, num2: float) -> bool:
    if num1 == num2:
        return True

    # For percentages and small numbers, lenient comparison
    if num1 < 1 and num2 < 1:
        return math.isclose(num1, num2, rel_tol=1e-2, abs_tol=1e-4)

    # For larger numbers, use the original comparison method
    dec_places1 = len(str(num1).split('.')[-1]) if '.' in str(num1) else 0
    dec_places2 = len(str(num2).split('.')[-1]) if '.' in str(num2) else 0
    round_to = min(dec_places1, dec_places2)
    rounded1 = round(num1, round_to)
    rounded2 = round(num2, round_to)

    if rounded1 == rounded2:
        return True
    return math.isclose(num1, num2, rel_tol=1e-2, abs_tol=1e-2)


def extract_numeric(value: str) -> Union[float, None]:
    value = value.replace(',', '').replace('$', '')
    match = re.search(r'(\d*\.\d+|\d+\.?\d*)%?', value)
    # @martini: ground truths are in % now, no need for this
    # if '%' in value: return float(num_str) / 100
    return float(match.group(1)) if match else None

Nine flaws holding us back.

  1. Fixation on single-shot text-to-SQL
  2. Low diversity of failure modes
  3. Insufficiently curated datasets
  4. Inaccurate grading
  5. Grounded in academia, not industry
  6. Unrealistic environments
  7. Unrealistic tasks
  8. Fundamentally non-agentic tasks
  9. Stateless evaluation ‼
Fixation on single-shot text-to-SQL → Analytical reasoning, not syntax
Low diversity of failure modes → Diverse realistic gotchas
Insufficiently curated datasets → One handcrafted, purpose-built dataset
Inaccurate grading → Calibrated LLM-judge grading
Grounded in academia, not industry → Maintained by someone in the field
Unrealistic environments → Rich, complex execution environment
Unrealistic tasks → Realistically complex analytics work
Fundamentally non-agentic tasks → Genuinely agentic tasks

G-02

G-07

ARR ≠ MRR × 12. Annual plans get two months free. Naive rollups overstate by ~17%.

G-09

Attribution inflation. Every platform claims the same conversions. The board deck uses the inflated numbers.

G-11

Undocumented status codes. Status integer 3 means “paused,” not “active.” Nobody documented it.

G-14

Cents vs dollars. Stripe stores cents; orders are dollars. Joins that forget look almost right.

G-23

The modeling layer is a trap too. Some dbt marts double-count refunds; stakeholders cite stale views.

query_warehouse:

    SELECT DATE_TRUNC('month', created_at) AS month,
           CASE WHEN channel IN ('mobile','mobile_app') THEN 1 ELSE 0 END AS is_mobile,

    WHERE is_valid_order AND NOT COALESCE(is_employee_discount, false)

  GROUP BY 1 ORDER BY 1;
